## שינוי קריטי במבנה ה-custom_groups

⚠️ **עדכון חשוב לגבי group_codes:**

בקובץ הכרטסת (ברון_2024.xlsx), כל חשבון שייך ל-group_code (600, 700, 702...).
אבל בדוח הרווח והפסד הזה, הסיווג הוא **לפי מפתח חשבון בודד**, לא לפי group_code.

למשל:
- מפתח 2505 "הוצ אחזקה" — בכרטסת הוא group_code 703. בדוח רו"ה הוא תחת "תפעול מפעל"
- מפתח 2830 "הוצ כשרות" — בכרטסת הוא group_code 703. בדוח רו"ה הוא תחת "תפעול מפעל"
- מפתח 2571 "שירותי כח אדם" — בכרטסת הוא group_code 704. בדוח רו"ה הוא תחת "קבלני משנה - לוגיסטיקה"

**המשמעות:** ה-custom_groups צריך לעבוד לפי **מפתח חשבון** (account code), לא רק לפי group_code.

### עדכון טבלת custom_groups:

```sql
-- במקום:
-- group_codes TEXT[] NOT NULL  -- ["700","701","702"]

-- צריך:
CREATE TABLE custom_groups (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES auth.users(id),
  name TEXT NOT NULL,
  display_order INTEGER DEFAULT 0,
  color TEXT,
  account_codes TEXT[] NOT NULL DEFAULT '{}',  -- ← מפתחות חשבון ספציפיים ["2505","2506","2830"]
  parent_section TEXT NOT NULL CHECK (parent_section IN (
    'revenue', 'cost_of_goods', 'operating', 'admin', 'finance', 'other'
  ))
);
```

### לוגיקת שיוך מעודכנת:

```
getAccountCategory(account, transaction, mode):

  // שכבה 1: override פנימי — גובר תמיד
  override = SELECT custom_group_id
             FROM account_classification_overrides
             WHERE account_id = account.id
  if override exists:
    → return override.custom_group

  // שכבה 2: חפש לפי account_code ב-custom_groups
  custom_group = SELECT * FROM custom_groups
                 WHERE account.code = ANY(account_codes)
                 AND user_id = $user_id
  if found:
    → return custom_group

  // שכבה 3: fallback — חשבון שלא שויך לאף קבוצה
  → return "אחר" (other)
```

## טיפול בחשבונות חדשים

כשמעלים קובץ כרטסת ויש בו חשבון שלא מופיע באף custom_group:

1. החשבון נכנס לקבוצת "אחר" (other)
2. מופיעה התראה למשתמש: "🆕 נמצאו 3 חשבונות חדשים שלא שויכו — לחץ לסווג"
3. המשתמש יכול לגרור אותם לקבוצה הנכונה במסך "מפתח סיווג פנימי"
