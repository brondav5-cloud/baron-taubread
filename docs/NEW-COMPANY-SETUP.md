# הקמת חברה חדשה / New Company Setup

מדריך להקמת חברה שנייה (או ראשונה) במערכת Bakery Analytics.

---

## שלב 1: שינוי הנתונים בסקריפט

1. פתח את הקובץ `supabase/scripts/create_new_company.sql`
2. **שנה רק את 3 הדברים הבאים:**
   - `'מאפיית שמש'` → שם החברה שלך (בעברית)
   - `'mafia-shemesh'` → מזהה ייחודי באנגלית (למשל `bakery-cohen` או `lechem-tov`)
   - כל שאר המופעים של `'mafia-shemesh'` בסקריפט – **חייבים להיות אותו מזהה**

**דוגמה לחברה "מאפיית כהן":**
- שם: `'מאפיית כהן'`
- slug: `'bakery-cohen'`

**טיפ:** השתמש בהחלפה (Ctrl+H): החלף את `mafia-shemesh` ב־`bakery-cohen` בכל הקובץ.

---

## שלב 2: הרצת הסקריפט

1. היכנס ל-**Supabase Dashboard** → **SQL Editor**
2. העתק את כל תוכן הקובץ (מהשלב א עד סוף שלב ד)
3. הדבק בחלון ה-SQL
4. לחץ **Run** (או Ctrl+Enter)
5. אם לא מופיעות שגיאות – החברה נוצרה

---

## שלב 2: יצירת משתמש מנהל ראשון

יש שתי דרכים:

### א. דרך Supabase Dashboard (מומלץ)

1. **Authentication** → **Users** → **Add user** → **Create new user**
2. הזן אימייל וסיסמה (למשל `admin@mycompany.com`)
3. סמן **Auto Confirm User** (או שלח אישור אימייל)
4. לחץ Create
5. **העתק את ה-UUID** של המשתמש מהטבלה (עמודה `id`)

### ב. דרך דף ההתחברות (אם קיים Sign Up)

אם יש כפתור "הרשמה" באפליקציה – המשתמש יכול להירשם בעצמו.  
**אבל:** הוא עדיין צריך שורה ב-`public.users` עם `company_id` ו-`role=admin`.  
זה מחייב הרצת ה-`INSERT` ידנית אחרי ההרשמה.

---

## שלב 3: קישור המשתמש לחברה

ב-**SQL Editor** הרץ (החלף את הערכים):

```sql
INSERT INTO users (id, company_id, email, name, role)
VALUES (
  'xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx'::uuid,  -- UUID מהשלב 2
  (SELECT id FROM companies WHERE slug = 'company-a'),  -- או company-b
  'admin@mycompany.com',
  'שם המנהל',
  'admin'
);
```

---

## שלב 4: התחברות והעלאת נתונים

1. המשתמש נכנס לאפליקציה עם האימייל והסיסמה
2. הוא יראה את הדשבורד של החברה שלו (ריק)
3. עבור ל-**העלאת נתונים** והעלה קובץ Excel
4. לאחר ההעלאה – החנויות, מוצרים והמדדים יופיעו

---

## הוספת משתמשים נוספים

אחרי שהמנהל מחובר:

1. **הגדרות** → **משתמשים** → **הוסף משתמש**
2. מלא שם, אימייל, תפקיד והרשאות
3. המשתמש החדש יקבל אימייל (או סיסמה זמנית) ויוכל להתחבר

**הערה:** רק `admin` ו-`super_admin` יכולים להוסיף משתמשים.

## סיכום: שתי חברות

| שלב | חברה א | חברה ב |
|-----|--------|--------|
| 1 | הרצת סקריפט עם `slug = 'company-a'` | הרצת סקריפט עם `slug = 'company-b'` |
| 2 | יצירת משתמש auth למנהל א | יצירת משתמש auth למנהל ב |
| 3 | INSERT ל-users עם company_id א | INSERT ל-users עם company_id ב |
| 4 | התחברות והעלאת נתונים | התחברות והעלאת נתונים |

כל משתמש רואה **רק** את הנתונים של החברה שלו – ההפרדה מבוססת על `company_id`.
